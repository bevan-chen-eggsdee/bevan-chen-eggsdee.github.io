    • Describe what the DOM is and an example of how you might interact with it.

  • The DOM refers to the Document Object Model which is the interface provided by the web browser representing a tree-like structure of a website. Each tree branch corresponds to a part of the document such as the elements, attributes, or text content. With the structure provided by the DOM, Javascript can be used to manipulate and interact with the document. An example of this can be sitting at a desk with a bunch of sticky notes each representing different elements of the webpage such as text, colour, size, etc. Each of these elements can represent the selection of an element, changing of contents, adding or removing elements, and responding to interactions.

    • Explain the difference between accessing data from arrays and objects.

  • Arrays refer to the ordered collection of similar items by numerical indices, and objects refer to the key-value pairs by unique identifiers. Arrays are more ideal for storing lists of items of the same type, typically numbers and strings. Objects are more ideal for storing entities with properties or attributes. They’re more flexible because keys can be more descriptive compared to arrays which are just numerical indices.

    • Explain what functions are and why they are helpful.

  • Functions are blocks of code that can be defined once and executed multiple times in code. They can hold a set of instructions that perform specific tasks or calculations and are capable of accepting parameters (arguments) and returning results. Functions are a building block for Javascript, they enable the reuse of code, abstraction, organisation, and many other things. The reusability of code allows for simplicity and less confusion with code due to repetition in different areas.
