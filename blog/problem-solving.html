PROBLEM SOLVING

When I was blocked on a problem

  • The problem that I was blocked on was the ‘manipulating-objects’ kata part where I couldn’t figure out why the spouse keys kept failing. I had completed all the other parts which were passing fine, but for some reason, the spouse key was failing whatever I did. I used the rubber ducky method going over each part one by one to understand what’s going on and why that specific part is failing. I felt so lost, had no idea what I was failing when it looked perfectly fine. Turns out it was just a simple grammatical error causing it to fail over and over. I learned to really pay attention to what I’m writing, even simple spelling or grammatical mistakes are going to cause problems.

    • Elegantly solved a problem

  • Working on the fizzbuzz kata, I was getting an error with the FizzBuzz, Buzz, and Fizz where the values were returning the incorrect string so they were failing when it was being tested. I used the rubber ducky method again to look through the code and explain which parts did what and then found that I had assigned the incorrect strings therefore leading to failed testing. Feeling pretty silly again, same as the manipulating-objects kata, simple mistakes where I hadn’t looked over properly. Having a proper re-read and explaining each part of code helped me find the mistake as from a first glance or just a glance over the code there doesn’t appear to be any problems. But once all the work is explained to yourself, you understand it better.
